Goals Reference
¶

This page documents goals, actions that can be taken on build targets. A goal identifies some high-level goal that you wish to accomplish, such as compiling your code.

A goal depends on other goals that must happen first; for example, Pants must compile code before it can run tests.

Each goal is implemented by one or more Tasks. For example, the ``JavaCompile`` task compiles Java code, and its installed in the ``compile`` goal.

Goals and Tasks summary

  • bench - None
  • binary - Create a runnable binary.
  • bootstrap - Bootstrap tools needed for building.
  • builddict - None
  • buildlint - None
  • bundle - Create an application bundle from binary targets.
  • changed - Print the targets changed since some prior commit.
  • check_published_deps - Find references to outdated artifacts.
  • clean-all - Clean all build output.
  • clean-all-async - [deprecated] Clean all build output in a background process.
  • compile - Compile source code.
  • compile-changed - Compile changed targets.
  • dependees - Print the target's dependees.
  • dependencies - Print the target's dependencies.
  • depmap - Depict the target's dependencies.
  • detect-duplicates - Detect duplicate classes and resources on the classpath.
  • dex - None
  • doc - Create documentation.
  • eclipse - Create an Eclipse project from the given targets.
  • ensime - Create an Ensime project from the given targets.
  • filedeps - Print out the source and BUILD files the target depends on.
  • filemap - Outputs a mapping from source file to owning target.
  • filter - Filter the input targets based on various criteria.
  • gen - Generate code.
  • goals - List all documented goals.
  • idea - Create an IntelliJ IDEA project from the given targets.
  • imports - None
  • invalidate - Invalidate all targets.
  • jar - None
  • killserver - Kill the reporting server.
  • list - List available BUILD targets.
  • markdown - Generate html from markdown docs.
  • minimize - Print the minimal cover of the given targets.
  • ng-killall - Kill running nailgun servers.
  • path - Find a dependency path from one target to another.
  • pathdeps - Print out all paths containing BUILD files the target depends on.
  • paths - Find all dependency paths from one target to another.
  • provides - Print the symbols provided by the given targets.
  • publish - Publish artifacts.
  • repl - Run a REPL.
  • repl-dirty - Run a REPL, skipping compilation.
  • resolve - Resolve dependencies and produce dependency reports.
  • resources - None
  • roots - Print the workspace's source roots and associated target types.
  • run - Run a binary target.
  • run-dirty - Run a binary target, skipping compilation.
  • server - Run the pants reporting server.
  • setup-py - None
  • sign - None
  • sort - Topologically sort the targets.
  • targets - List target types and BUILD file symbols (python_tests, jar, etc).
  • test - Test compiled code.
  • test-changed - Test changed targets.
  • thrift-linter - Check thrift files for non-recommended usage patterns.

Create an Eclipse project from the given targets.

eclipse
¶

Implemented by pants.backend.jvm.tasks.eclipse_gen.EclipseGen

  • --nailgun-server list_type (Default: [u'//:nailgun-server']) Target specs for bootstrapping the nailgun-server tool.
  • --jar-tool list_type (Default: [u'//:jar-tool']) Target specs for bootstrapping the jar-tool tool.
  • --project-name str (Default: project) Specifies the name to use for the generated project.
  • --project-dir str Specifies the directory to output the generated project files to.
  • --project-cwd str Specifies the directory the generated project should use as the cwd for processes it launches. Note that specifying this trumps --eclipse-project-dir and not all project related files will be stored there.
  • --[no-]intransitive bool Limits the sources included in the generated project to just those owned by the targets specified on the command line.
  • --[no-]python bool Adds python support to the generated project configuration.
  • --[no-]java bool (Default: True) Includes java sources in the project; otherwise compiles them and adds them to the project classpath.
  • --java-language-level int (Default: 7) Sets the java language and jdk used to compile the project's java sources.
  • --java-jdk-name str Sets the jdk used to compile the project's java sources. If unset the default jdk name for the --java-language-level is used
  • --[no-]scala bool (Default: True) Includes scala sources in the project; otherwise compiles them and adds them to the project classpath.
  • --[no-]use-source-root bool Use source_root() settings to collapse sourcepaths in project and determine which paths are used for tests. This is usually what you want if your repo uses a maven style directory layout.
  • --[no-]infer-test-from-siblings bool When determining if a path should be added to the IDE, check to see if any of its sibling source_root() entries define test targets. This is usually what you want so that resource directories under test source roots are picked up as test paths.
  • --version str (Default: 3.6) The Eclipse version the project configuration should be generated for.

Common
¶

These command-line flags are available with all Pants goals.

  • --pants-workdir str (Default: <buildroot>/.pants.d) Write intermediate output files to this dir.
  • --pants-supportdir str (Default: <buildroot>/build-support) Use support files from this dir.
  • --pants-distdir str (Default: <buildroot>/dist) Write end-product artifacts to this dir.
  • --config-override str A second config file, to override pants.ini.
  • --[no-]pantsrc bool (Default: True) Use pantsrc files.
  • --pantsrc-files str (Default: [u'/etc/pantsrc', u'~/.pants.rc']) (Default: [u'/etc/pantsrc', u'~/.pants.rc']) Override config with values from these files. Later files override eariler ones.
  • --pythonpath str Add these directories to PYTHONPATH to search for plugins.
  • -t/--timeout int Number of seconds to wait for http connections.
  • -x/--[no-]time bool Times tasks and goals and outputs a report.
  • -e/--[no-]explain bool Explain the execution of goals.
  • -k/--[no-]kill-nailguns bool Kill nailguns before exiting
  • --[no-]ng-daemons bool (Default: True) Use nailgun daemons to execute java tasks.
  • -d/--logdir str Write logs to files under this directory.
  • -l/--level str (Default: info) Set the logging level.
  • -q/--[no-]quiet bool Squelches all console output apart from errors.
  • -i/--interpreter str Constrain what Python interpreters to use. Uses Requirement format from pkg_resources, e.g. 'CPython>=2.6,<3' or 'PyPy'. By default, no constraints are used. Multiple constraints may be added. They will be ORed together.
  • --no-colors bool Do not colorize log messages.
  • --no-lock bool Don't attempt to grab the global lock. This lock prevents two concurrent pants instances from stomping on each others data, so only use this if you know what you're doing.
  • --spec-excludes str (Default: ['<buildroot>/.pants.d']) Exclude these paths when computing the command-line target specs.
  • --exclude-target-regexp str Regex pattern to exclude from the target list (useful in conjunction with ::). Multiple patterns may be specified by setting this flag multiple times.
  • --[no-]read-from-artifact-cache bool (Default: True) Read build artifacts from cache, if available.
  • --read-artifact-caches list_type The URIs of artifact caches to read from. Each entry is a URL of a RESTful cache, a path of a filesystem cache, or a pipe-separated list of alternate caches to choose from.
  • --[no-]write-to-artifact-cache bool (Default: True) Write build artifacts to cache, if possible.
  • --write-artifact-caches list_type The URIs of artifact caches to write to. Each entry is a URL of a RESTful cache, a path of a filesystem cache, or a pipe-separated list of alternate caches to choose from.
  • --[no-]overwrite-cache-artifacts bool If writing to build artifacts to cache, overwrite (instead of skip) existing.
  • --[no-]print-exception-stacktrace bool Print to console the full exception stack trace if encountered.
  • --[no-]fail-fast bool When parsing specs, will stop on the first erronous BUILD file encountered. Otherwise, will parse all builds in a spec and then throw an Exception.
